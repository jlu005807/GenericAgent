# SOP 格式规范 v2.0

**本规范基于实际使用经验简化，定义 GenericAgent SOP 文件的实用格式,但是不需要太多的规范，尽量将施展的空间交给LLM**

---

## 1. 文件命名

```
<skill_name>_sop.md       # 技能类 SOP（如 browser_auto_sop.md）
<workflow_name>_sop.md    # 流程类 SOP（如 memory_cleanup_sop.md）
```

命名规则：
- 全小写，下划线分隔
- 后缀 `_sop.md`（与记忆/笔记文件区分）
- 简短明确，无需前缀（文件夹已区分领域）

---

## 2. 核心结构（4个部分）

```markdown
# 技能名称 SOP

触发条件 | 核心规则一句话

## 一、核心规则（必填）
硬性规则列表

## 二、快速用法/API（必填）
代码示例或API速查

## 三、工作流程（必填）
步骤列表，简洁明了

## 四、避坑指南（推荐，即可要可不要或者让LLM自动去补全）
常见陷阱和注意事项
```

---

## 3. 各部分要求

### 标题行（必填）
- `# 技能名称 SOP`
- 紧跟一行：触发条件 + 核心规则摘要
- 示例：`**触发**：3步以上任务 | **禁用**：1-2步简单任务直接做`

### 核心规则（必填）
- 列表形式，每条一行
- 重点：**禁用项** > **必须项**

### 快速用法/API（必填）
- Python代码块或API签名列表
- 能直接复制粘贴使用，方便LLM使用工具

### 工作流程（必填）
- 数字编号步骤
- 每步一个动作，不超过10步
- 尽量不要太过复杂，过于冗余的流程可以分为子SOP

### 避坑指南（推荐）
- 常见错误和解决方案,可以是你自己使用时出现的问题或者LLM自动补全的问题

---

## 4. 编写原则

1. **简洁优先**：能用一句话说清的不用两句话
2. **实用导向**：只写实际需要的内容，删除模板冗余
3. **代码可执行**：示例代码必须可直接运行

---

## 5. 不需要的内容

- ❌ YAML Front Matter（无需元数据头）
- ❌ 版本历史（无需追踪）
- ❌ 错误处理表格（用列表替代）
- ❌ 权限边界表格（用列表替代）
- ❌ 状态机/流程图（简单流程用列表）
